# WIK-151 — i18n

Soporte multilingüe del producto (`en` + `es`). Single source of truth:
`src/i18n/locales.ts`. Default locale: `en`.

## Selección de idioma en runtime

Cadena de resolución (ver `src/i18n/resolve-locale.ts`):

1. Cookie `NEXT_LOCALE` (seteada por el selector del footer).
2. `profiles.language` si hay user logueado.
3. `Accept-Language` del navegador.
4. `DEFAULT_LOCALE` (`en`).

## WhatsApp templates EN — submit a Meta (P5)

Meta trata cada template como un par `(name, language)`. Los templates ES
están aprobados; cada variante EN se registra como un template nuevo con
el mismo `name` + `language: "en"` y pasa por su propia review.

### 1. Submit

Requiere `KAPSO_API_KEY` y `WHATSAPP_WABA_ID` en el env.

```sh
# Dry-run primero (no llama a Meta, solo muestra los bodies)
npm run wa:templates:submit:dry

# Live submit — manda los 16 templates (8 ES ya aprobados + 8 EN nuevos).
# Los ES vuelven con "duplicate name" y se saltean; los EN entran a PENDING.
npm run wa:templates:submit
```

### 2. Esperar approval

Meta tarda **24-72h** en revisar UTILITY templates. Suele ser bastante
rápido (mismas reglas que las variantes ES, ya validadas). No hay nada que
hacer en este paso — el cron de la app sigue funcionando porque cae al
fallback ES (ver siguiente sección).

### 3. Verificar estado

```sh
npm run wa:templates:status
```

Lista cada template con su `status` actual (`PENDING`, `APPROVED`,
`REJECTED`). Cuando los 8 EN aparezcan `APPROVED`, el fallback deja de
dispararse en producción.

### 4. Fallback durante la ventana de aprobación

`sendKapsoTemplateWithFallback` (en `src/lib/whatsapp/index.ts`) llama
primero con el `languageCode` del recipient (`profile.language` coerced a
`Locale`, default `en`). Si Meta/Kapso devuelven error (caso típico:
template EN aún `PENDING`), reintenta con `"es"` y loguea un warning.
Mientras los EN no estén `APPROVED`:

- Recipients con `language: "en"` reciben el mensaje en ES — degrada
  gracefully pero el mensaje sí llega.
- Recipients con `language: "es"` no se ven afectados.

Una vez aprobados todos los EN, no hay que tocar nada — el primer intento
empieza a funcionar y el fallback nunca corre.

## Strings UI

Cada dictionary vive en `messages/<locale>.json`. Usar `next-intl`:

- Server components: `getTranslations({ locale, namespace })`.
- Client components: `useTranslations(namespace)`.

Mantener voseo en ES (`tenés`, `podés`); inglés neutro en EN.
